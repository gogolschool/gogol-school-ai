# /collect-stats

## Что делает

Снимает метрики активности из нужного источника и пишет их в поля-метрики карточки 🎯 Активности. Формулы (`CR клики`, `CR оплаты`, `ROAS`) не трогает. Ничего не выдумывает: чего не хватает (например, скрин Instagram или доступ к внешнему TG) — просит.

## Как работает

1. **Загрузи актуальный мастер-промт из Notion** (рабочий MCP `c2755fd9-…`, не личный — память `feedback_notion_workspace_routing`): page `388612c762af81619bcec028061e6b64` («📊 Как вносить статистику в Активности»). Действуй строго по нему.
2. **Определи режим:**
   - «Сними статистику по этой активности: {URL}» — одна карточка.
   - «Собери статистику за неделю» — все карточки 🎯 Активности (`collection://34a612c7-62af-80b6-85e6-000bdf26af93`) со статусом «опубликовано», пустыми метриками, у которых с даты публикации прошло **≥3 дней** (метрики к этому моменту устаканиваются).
3. **По полю `Тип канала` определи источник** (матрица §3 промта) и сними метрики (§4).
4. **Запиши в поля-метрики** карточки (`notion-update-page`). Формулы `CR клики / CR оплаты / ROAS` НЕ заполняй — считаются сами.
5. **Переведи статус** карточки в «посчитано».

## Матрица: Тип канала → откуда брать

| Тип канала | Метрики | Источник |
|---|---|---|
| `mail` | Охват, Open Rate %, Click Rate %, Визиты | **Unisender** + **Я.Метрика** |
| `bot-*` | отправлено, Визиты | данные бота + **Я.Метрика** |
| `tg-*` (свои) | Охват, Взаимодействия, ER/ERR, Визиты | **TGStat** / **Telegram** + **Я.Метрика** |
| посев (коллаборация) | + Бюджет, Клики | **TGStat** + чат с админом канала |
| `inst-*` | Охват, Взаимодействия, (сторис) клики | **Instagram Insights** (скрин→Claude) |
| Оплаты / Лиды (любой) | оплаты, заявки | **Я.Метрика** (цели) / **CRM** |

## Что автоматизировано, а что вручную

- **`mail`** — ✅ полностью: `unisender` MCP `get_campaign_stats` (отдаёт reach / open_rate_pct / click_rate_pct) + `get_visited_links`; визиты — Я.Метрика по UTM.
- **Визиты / Клики / Достижения целей / Оплаты по UTM (любой канал)** — ✅ `yandex-metrika` MCP `get_utm_report` (фильтр по метке активности), счётчик `58973071`.
- **`tg-*` свои каналы** — ⚠️ через `telegram_gogolschool` MCP (на момент написания сессия требует переавторизации); пока — вручную из TGStat.
- **Внешние посевы (TGStat)** — связки нет: Охват/ER из TGStat вручную, бюджет — из чата с админом канала.
- **`inst-*`** — ⚠️ без API: сотрудник шлёт **скриншот** Insights (пост: Охват/Взаимодействия; сторис: через Архив → Охват/клики), Claude считывает цифры с картинки и пишет в карточку.

## Критические правила (из промта)

1. **Когда снимать:** через **≥3 дня** после публикации (email и Метрика догоняют ~48 ч). Работаем только с таблицей 🎯 Активности; «💌 Рассылки» не используем.
2. **Дробление Unisender:** аудитория **<1000** → один список = одна рассылка, цифры напрямую. **>1000** → список дробится (10–11 рассылок): проценты **усредняют** `(OR₁+…)/N`, абсолютные **суммируют**.
3. **Open Rate:** `mail` = прочитано ÷ доставлено (Unisender); `tg` = Охват ÷ подписчики.
4. **Маппинг в поля:** Охват ← доставлено (mail) / просмотры (tg); Open Rate % ← прочитано %; Click Rate % ← переходы %; Визиты из Метрики ← «Визиты»; Клики ← переходы по метке; Достижения целей ← «Достижения целей»; Оплаты шт/₽ ← целевое действие.
5. **Несколько UTM:** в поле `UTM` — одна ссылка; остальные — **в комментарии** к карточке.
6. **Не трогать формулы:** `CR клики`, `CR оплаты`, `ROAS`.
7. **Формат процентов (КРИТИЧНО):** поля `Open Rate, %` и `Click Rate, %` в Notion — формата «процент», хранят **долю**: 34% = `0.34`, 7.7% = `0.077`. Unisender отдаёт проценты (напр. `26.9`) → перед записью **дели на 100** (`0.269`). НИКОГДА не пиши целое число процентов в эти поля — `26.9` отобразится как `2690%`.
8. **Не перезаписывай молча:** заполняй только **пустые** поля-метрики. Если поле уже заполнено — покажи старое и новое значение и **спроси**, обновлять ли. Без подтверждения существующие цифры не трогай.
