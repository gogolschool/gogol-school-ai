# /add-utm

## Что делает

Собирает UTM-ссылку из полей карточки 🎯 Активности и пишет её в поле `UTM`. Только это поле — остальное в карточке не трогает. Ничего не угадывает: чего не хватает (URL, кампания) — спрашивает.

## Как работает

1. **Загрузи актуальный мастер-промт из Notion** (рабочий MCP `c2755fd9-…`, не личный — память `feedback_notion_workspace_routing`): page `367612c762af81e384a6cd7c86154dc9` («🔗 UTM-разметка»). Действуй строго по нему — там таблицы source/medium, форматы campaign, валидация.
2. **Определи режим:**
   - «Размечь UTM в этой карточке: {URL}» — одна карточка.
   - «Размечай UTM на эту неделю» — все карточки 🎯 Активности (`collection://34a612c7-62af-80b6-85e6-000bdf26af93`), у которых `Дата` попадает в текущую неделю (ПН–ВС).
3. **Собери 4 параметра** из полей карточки (см. ниже). Не хватает обязательного — спроси, не выдумывай.
4. **Собери ссылку** `{URL}?utm_source=..&utm_medium=..&utm_campaign=..&utm_content=ACT-{ID}`, прогони валидацию.
5. **Запиши в поле `UTM`** карточки (`notion-update-page`, свойство `UTM`). Для недельного прогона покажи сводку собранных UTM перед записью.

## Откуда что берётся

| Параметр | Источник | Правило |
|---|---|---|
| `base_url` | поле `URL` карточки | взять как есть. Пусто → взять часть до `?` из текущего поля `UTM` или первую ссылку `gogolschool.ru/...` в теле; если и там нет — **спросить** |
| `utm_source` | `Тип канала` (префикс до дефиса) | `mail`→mail, `bot-*`/`бот-*`→bot, `tg-*`→telegram, `inst-*`→instagram, `vk-*`→vk, `site-*`→site, `внешний-СМИ`→smi, `внешний-упоминание`→external, `коллаборация`→collab, `бартер`→barter, `мероприятие`→event, `пуш-карта`→push, `pinterest`→pinterest |
| `utm_medium` | `Тип канала` (суффикс) / `Площадка` | mail→mail; bot-абонементов→subs; бот-главный→main; tg-mtnr→mtnr; tg-b2b→b2b; tg-канал-alumni→alumni; inst-рилс→reels/пост→post/сторис→stories; tg-comm/внешний-*→слаг канала из «Площадка» (пусто → из заголовка; не вышло — спросить) |
| `utm_campaign` | `Name` связанной `🌵 Календарь маркетинг` | транслит в kebab-ASCII (нижний регистр, спецсимволы→дефис). Связи нет → **спросить кампанию**, не угадывать по заголовку |
| `utm_content` | поле `ID` карточки | `ACT-{число}`. См. критическое правило ниже |

## Критические правила (выверено на живых карточках 2026-06-30)

1. **`ID` уже приходит как `ACT-305`** (строка с префиксом), а не голое число. `utm_content` = значение поля **как есть** — НЕ добавляй второй `ACT-` (иначе `ACT-ACT-305`). Если поле `ID` пустое — `utm_content` пропустить, остальное собрать.
2. **На реальных карточках `URL` и `🌵 Календарь маркетинг` часто ПУСТЫЕ.** По штатному процессу их заполняет маркетолог до запуска. Если пусто — действуй по фолбэкам выше (URL — из тела/старого UTM) и **спрашивай кампанию**; не выдумывай ни ссылку, ни кампанию.
3. **Берём НОВЫЙ промт, не старую «опись полей».** В описи (`367612c7…5da1`, май) сказано «utm_content не добавляется» и campaign форматами A/B/C слитно — это устарело: `ID` в базе уже есть, текущий промт собирает campaign из `Name` Календаря в kebab и добавляет `utm_content`.
4. **Валидация перед записью** (§4 промта): `base_url` на `gogolschool.ru`; `utm_source` из допустимого набора; `utm_medium`/`utm_campaign` — kebab-ASCII, не пустые; `utm_content` = `ACT-{число}`; **никакой кириллицы, заглавных, пробелов** в значениях.
5. **Пишем только поле `UTM`.** В остальные поля карточки скилл не лезет (атрибуция полей — память по описи; UTM — единственное, что ставит автоматика).
6. **`utm_source` платформы, не канал.** Для tg-каналов source=`telegram`, конкретный канал идёт в `medium` (не `utm_source=tg-mtnr`).
