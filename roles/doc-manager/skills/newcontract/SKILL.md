# /newcontract

## Что делает

По заявке в `crm.actions_for_contacts` генерирует договор оказания услуг (PDF/DOCX) через action в схеме `reports`. Поддерживает варианты:

- **Договор оказания услуг** — `reports.generate_and_export_contract_document(id, ext)`
- **Договор с допсоглашением** — `reports.generate_and_export_contract_document_for_agreement`
- **Договор от абонемента** — `reports.generate_and_export_contract_document_from_subscription`
- **Счёт-оферта** — `reports.create_offer_invoice(...)`

## Как работает

1. Уточнить у пользователя:
   - **ID заявки** (`crm.actions_for_contacts.id`) — или искать по имени студента + продукту
   - **Тип документа**: основной договор / допсоглашение / абонемент / счёт-оферта
   - **Формат**: `.pdf` (по умолчанию) или `.docx`
2. Загрузить актуальный мастер-промпт «Создание договора» из Notion prompt library (page `6f0c0859c8ab46e2838107aeb3b4726c` — корень, искать `🤖 Промт: Создание договора`).
3. Показать пользователю что собираюсь сделать (ID заявки, студент, продукт, тип, формат) → подтверждение.
4. Запустить `run_action` с нужным action_id и аргументами.
5. Вернуть ссылку на сгенерированный документ.

## Важно

- **`agreeText` использует разбивку 30/50/20% по модулям** + указание `refund@gogol.school` для возвратов. Если автогенерация даёт что-то другое — проверять глазами.
- **Перед вручением документа клиенту** — открыть PDF и проверить: ФИО, паспортные данные (для самозанятых), даты, сумма, расписание.
- Если заявка ещё не в нужном статусе (например, не оплачена) — уточнить у пользователя, точно ли создавать договор сейчас.
