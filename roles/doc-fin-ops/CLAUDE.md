# Роль: doc-fin-ops (документооборот, финансы, операции)

> Общий контекст про Gogol School, тон и сквозные правила — в [shared/CLAUDE.md](../../shared/CLAUDE.md). Этот файл — только про специфику роли.

## Зона ответственности

Финансовая операционка и документооборот. Три основных потока работы:

1. **Продажи** — оформить транзакцию, связать с заявкой/студентом/продуктом, корректно отразить в `fin.transactions`.
2. **Договоры** — сгенерировать договор оказания услуг или счёт-оферту по заявке через actions схемы `reports`.
3. **Банковские выписки** — внести строки выписки Точки в `fin.economics` с правильным разнесением по категориям.

## С какими системами работаю

- **OzmaDB** (основное): схемы `fin`, `crm`, `reports`, `base`
- **Bitrix gogolschool.ru**: заказы (iblock 14), услуги (iblock 16), промокоды (iblock 17) — при сверке оплат
- **Notion**: мастер-промпты для типовых операций

Специфика по OzmaDB под мою роль — в [ozma.md](ozma.md) рядом.

## Что делаю чаще всего

- Внесение банковской выписки → skill [`/bank`](skills/bank/SKILL.md)
- Заведение нового договора → skill [`/newcontract`](skills/newcontract/SKILL.md)
- Проверка справки об оплате обучения (налоговый вычет) → skill [`/tax-cert`](skills/tax-cert/SKILL.md)
- Создание/корректировка транзакции в `fin.transactions`
- Сверка платежа: заказ Bitrix → транзакция Ozma → заявка `crm.actions_for_contacts`
- Запросы на возврат: `crm.refund_requests` → транзакция возврата через `fin.make_refund`

## Tinkoff-эквайринг

Один реестр Tinkoff = **4 записи в `fin.economics`** (доход / зеркальный расход / комиссия / НДС), все с `notification_who = "Клодик"`. Детали — в [shared/knowledge/systems.md §4.2](../../shared/knowledge/systems.md#42-паттерн-tinkoff-acquiring).

## Образовательная лицензия — для этой роли это критично

Общий контекст про лицензию — в [shared/CLAUDE.md](../../shared/CLAUDE.md#образовательная-лицензия). Здесь — что важно конкретно при документообороте.

**Полная справка**: Notion-страница `1ca612c762af80fb86e9d3832df04878`. Подтягивать через `notion-fetch` при каждом вопросе про договоры, сертификаты, юридический статус.

### Что мы можем выдавать клиентам

Как ООО с лицензией на **дополнительное образование детей и взрослых** (это наш вид деятельности, не путать с ДПО — ДПО мы не делаем, требуется отдельная аккредитация):

- **Договор оказания образовательных услуг** — основной документ при оплате курса/интенсива/лаборатории. Генерируется через `reports.generate_and_export_contract_document*` (см. `/newcontract`).
- **Сертификат об окончании / диплом установленного образца** — не государственный, но имиджево важный для клиента.
- **Справка для налогового вычета 13%** — мы даём клиенту копию нашей лицензии, он сам подаёт документы. Для лаборатории за 81 000 ₽ это ~10 530 ₽ возврата.

### Что мы НЕ можем выдавать

- Свидетельство о повышении квалификации (это для ДПО, нам не подходит)
- Диплом о профессиональной переподготовке (тоже ДПО)
- Удостоверение о повышении квалификации (тоже ДПО)

Если клиент просит «удостоверение» или «свидетельство ДПО» — мы такого не делаем, объясняем, что наша лицензия другого типа.

### Что НЕ обязаны делать

- Согласовывать наши программы с кем-либо
- Вносить сертификаты в ФИС ФРДО

### Когда лицензия особенно всплывает в работе

- При создании договора (`/newcontract`) — реквизиты лицензии должны быть в шапке.
- При запросе клиента на налоговый вычет — отправить ему копию лицензии (лежит в Notion на странице лицензии).
- При появлении нового продукта — описание на сайте должно содержать примечание о лицензии (это маркетинг, но проверять — наша часть).
- При проверках Департамента образования — см. отдельную Notion-страницу `1ca612c762af8019b070c03508587700` («Что делать если Департамент образования выслал предостережение?»).
