# /bank

## Что делает

1. **Внесение выписки**: берёт банковскую выписку Точки (CSV или текстом) и разносит строки в `fin.economics` OzmaDB: правильные категории, контрагенты, теги, пометка `notification_who = "Клодик"`.
2. **Проверка остатков эквайринга**: сводит дельты в `fin.report_fin_balances_new` (account_type=6) и находит причины расхождений (фантомы, дубли, неверные даты возмещения, дубли НДС, авансовый счёт, возврат не на тот счёт). Цель — дельта 0 каждый месяц.

## Как работает

1. Загружает актуальный мастер-промпт из Notion:
   - внесение выписки → page `320612c762af81ceb53bcfc18fd27514`;
   - **проверка остатков эквайринга → page `380612c762af81c79499c5eb239c63ff`**.
2. Если выписка ещё не приложена — просит её у пользователя (файл / текст / ссылка).
3. Действует строго по мастер-промпту. Перед массовой вставкой — показывает превью первых 3–5 строк на проверку.
4. После вставки — отчитывается: сколько записей внесено, какие категории, общие суммы.

## Важно

- Tinkoff-эквайринг: с **11.02.2026** банк пишет «в т.ч. НДС» → комиссия = **3 записи** (доход / зеркальный расход / полная комиссия с НДС внутри). Отдельную строку НДС НЕ заводить — это давало задвоение (исправлено). До 11.02 было «Ставка НДС» (НДС отдельно) — там 4 записи, легитимно. Детали — в мастер-промпте.
- Удаление записей: сначала чистить зависимости в `fin.pl_report_final` (и `subscription_info`/`requests_for_subscriptions` для доходных проводок), потом основную запись.
- При сверке остатков: дата дохода = дата **возмещения** (как в банке), реестр матчить по «Дате перечисления», не «операции».
- Не править вручную — править мастер-промпт в Notion, чтобы изменения попали ко всем.
