# /add-product

## Что делает

Заводит новую строку в Notion-таблице **«🗂️ Таблица с продуктами»** (data source `collection://9728f62f-5c91-493b-8818-3bcdedae0a82`) из данных менеджера по расписанию. Источник — свободный текст в чате и/или ссылка на Google Doc с расписанием и мастерами. Заполняет **только блок оунера/расписания**; поля маркетинга и финансов не трогает.

## Как работает

1. **Загрузи актуальный мастер-промпт из Notion:** page `387612c762af81988be7c8edd61542ed`
   (https://www.notion.so/387612c762af81988be7c8edd61542ed). Действуй строго по нему — там список полей, справочники select-значений, правила маппинга, значения по умолчанию и самопроверка.
   > Notion берём через **рабочий** MCP (`c2755fd9-…`), не личный (см. память `feedback_notion_workspace_routing`).
2. Собери данные: если дана ссылка на Google Doc — открой его (MCP `google_docs`) и вытащи расписание/мастеров/даты/место; добавь детали из свободного текста; сведи воедино.
3. **Покажи сводку** (поле → значение), отдельно перечисли незаполненные поля, спорные допущения (выведенный сезон/статус) и конфликты «текст vs Google Doc». Дождись подтверждения.
4. Создай новую страницу в data source `collection://9728f62f-…`, выполни самопроверку из промта, отчитайся ссылкой на строку и списком заполненных полей.

## Критические правила

- **Только блок оунера/расписания.** Поля маркетинга (Название для сайта, Описание, Тема) и финансов (Стоимость, EB, Изменение стоимости, Ссылка в CRM, Ссылка на оплату, Срок действия ссылки, Нужна ли рассрочка, Промокоды) **не заполняй** — их вносят другие роли.
- **Точные имена полей.** У части полей в схеме в конце имени стоит символ табуляции (`Первое занятие (дата)\t`, `Расписание\t`). Перед записью сверь имена через `fetch` коллекции. Для дат — разворачивай в `date:<имя поля>:start` (и `:end`, `:is_datetime` при необходимости).
- **Select строго из справочников.** Направление, Статус, Сезон, Тип продукта — только значения из мастер-промта; не выдумывай новых опций.
- **`Кол-во необходимых ОУ` не оставлять пустым** — если ОУ не нужны, ставь `0`.
- **Подтверждение перед записью** — всегда сводка и согласие менеджера.
- **Чего нет в данных — не выдумывай** (кроме правила про ОУ).

## Что НЕ делает

- Не заводит продукт в OzmaDB/CRM и не создаёт ссылку на оплату на сайте — для этого `/newproduct`.
- Не создаёт Открытые уроки как sub-items (отдельный шаг таблицы).
