# /add-calendar

## Что делает

Раскладывает занятия программы по общему Google-календарю **«GOGOL school»** (`gschool182@gmail.com`): из расписания (даты + время) создаёт **по одному событию на каждое занятие**. Источник данных — свободный текст, строка «🗂️ Таблицы с продуктами» или Google Doc с расписанием/мастерами.

## Как работает

1. **Загрузи актуальный мастер-промпт из Notion:** page `387612c762af81cfb20ad67aaf962889`
   (https://www.notion.so/387612c762af81cfb20ad67aaf962889). Действуй строго по нему — там целевой календарь, конвенция заголовков, правила времени, формат событий и порядок.
   > Notion берём через **рабочий** MCP (`c2755fd9-…`), не личный (см. память `feedback_notion_workspace_routing`).
2. Вызови `get-current-time`, собери данные программы (текст / строка таблицы / Google Doc через MCP `google_docs`).
3. Раскрой расписание в список занятий (дата + время начала/окончания каждого).
4. **Проверь дубли** в календаре (`list-events` по диапазону дат программы).
5. **Покажи сводку** будущих событий и дождись подтверждения.
6. Создай события (`create-event` / `create-events` bulk) в календаре `gschool182@gmail.com`, отчитайся числом и ссылками.

## Критические правила

- **Календарь:** `gschool182@gmail.com` («GOGOL school»). Часовой пояс `Europe/Moscow`.
- **Время — московское, без UTC-сдвига.** В отличие от OzmaDB (там вычитают 3 часа) здесь передаём локальное время как есть с `timeZone: "Europe/Moscow"`.
- **Одно событие на занятие.** Заголовок: `{ТИП/код} {Название} {Фамилии мастеров} [Зал N]`; онлайн-программы — префикс «Онлайн» и `colorId: 9`. Первое занятие длительного продукта (от 2 занятий) — с пометкой «ЗАПУСК» в конце (как `HEAD 11 ЗАПУСК`).
- **Подтверждение перед записью** — создание событий в общем календаре это внешнее действие, всегда сводка и согласие.
- **Идемпотентность:** перед созданием сверяйся с существующими событиями; дубли не создавай. Ничего не удаляй/не правь без явной команды.

## Что НЕ делает

- Не заводит продукт в таблицу Notion — для этого `/add-product`.
- Не заводит продукт в OzmaDB/CRM и на сайт — для этого `/newproduct`.
- Не ведёт брони/аренды залов — только занятия программ.
