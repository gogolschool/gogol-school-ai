# Инструкция-промт: создать Google-расписание встреч («запись на зумы»)

**Триггер:** Белла пишет «сделай расписание зумов», «создай график для записи», «настрой страницу бронирования встреч», «appointment schedule» и т.п.

## Важное ограничение
Google Calendar **не отдаёт «Графики встреч» (Appointment schedule) через API** — ни создать, ни прочитать программно нельзя. Поэтому график создаётся **вручную через браузер** (Chrome MCP) в залогиненном аккаунте **fabell1407@gmail.com** на `calendar.google.com` (только веб-версия на компьютере). Через календарный MCP видны только уже состоявшиеся брони (как обычные события), сам пустой график — нет.

## Что спросить у Беллы ПЕРЕД началом
1. **Название** графика (напр. «Расписание зумов»).
2. **Длительность** встречи (напр. 60 мин).
3. **Диапазон дат доступности** — с какого по какое. ⚠️ Часто главная ошибка: окно начинается слишком поздно, и на ближайшие дни слотов нет. Уточнить, что записи нужны уже с ближайших дней.
4. **Дни недели и часы** приёма + **часовой пояс** (обычно Москва GMT+3; на странице посетитель видит свой пояс).
5. **Тип встречи / место:** Google Meet / ссылка Zoom / телефон / оффлайн.
6. Доп. параметры при желании: буфер между встречами, макс. встреч в день, минимальное/максимальное уведомление о записи, вопросы в форме бронирования.
7. **⭐ Какие календари проверять на занятость** — ГЛАВНЫЙ пункт (см. ниже).

## ⭐ Ключевая ловушка (из-за неё в прошлый раз график был «пустой»)
В настройках графика есть блок **«Проверять доступность в календарях» / «Calendars checked for availability»**. Если там отмечены несколько календарей, слоты, пересекающиеся с ЛЮБОЙ занятостью в них, скрываются автоматически. У Беллы календари плотно забиты зумами → почти все слоты вырезаются, страница выглядит пустой.

**Правило:** оставлять в проверке **только реально блокирующие** календари (обычно один основной). Информационные/чужие — **Саша Работа, Праздники России, GS Труппа, Пример абонементы** и т.п. — из списка **убирать**. Либо задавать чёткие рабочие часы в заведомо свободных окнах.

## Шаги в интерфейсе (веб, компьютер)
1. Выбрать браузер через Chrome MCP (если подключено несколько — спросить какой), открыть `calendar.google.com`, убедиться что залогинен fabell1407.
2. Кнопка **«Создать» → «График приёма встреч»** (Create → Appointment schedule).
3. Ввести **название** и **длительность**.
4. Задать **общую доступность**: дни недели + часы; проверить **диапазон дат** (дата начала — не в будущем без причины).
5. Раскрыть настройки, выставить: место встречи (Meet/Zoom/телефон), буфер, лимиты, окно уведомления.
6. **Настройки → «Проверять доступность в календарях»** — оставить только нужные календари (см. ловушку выше).
7. **Сохранить**.
8. Открыть **«Открыть страницу бронирования»** / скопировать ссылку на бронирование.

## Проверка результата (обязательно)
Открыть полученную ссылку через браузер как посторонний (в идеале инкогнито) и убедиться, что:
- страница грузится, есть заголовок и длительность;
- на **ближайшие** нужные дни есть свободные слоты (не «Нет доступных вариантов»);
- часовой пояс и часы совпадают с ожидаемыми.
Отдать Белле ссылку.
