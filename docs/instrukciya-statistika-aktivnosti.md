## 1. Что это за таблица

Каждая строка = одна маркетинговая **активность** (пост, сторис, рассылка, посев, мероприятие).
Поле **`Тип канала`** определяет, **из какого источника** снимать метрики.

> **Когда собирать:** статистику снимаем **через ~3 дня после публикации** поста/письма/сториса —
> к этому моменту метрики (охваты, открытия, переходы, визиты) устаканиваются. Раньше не стоит:
> данные ещё «докручиваются» (особенно email и Метрика догоняют ~48 ч).
>
> Работаем **только с таблицей 🎯 Активности**. Таблица «💌 Рассылки» для статистики **не используется**.

## 4. Пошагово по каждому источнику

### 4.1. Unisender — для `mail`
1. `app.unisender.com` → **Рассылки → История**.
2. Найти рассылку по теме/дате. Видны 4 показателя: **отправлено · доставлено · прочитано (%) · переходы (%)**.
3. Перенос в Активности:
   - **Охват** ← доставлено
   - **Open Rate, %** ← прочитано (%)
   - **Click Rate, %** ← переходы (%)

> ⚠️ **Заковыка с дроблением списков:**
> - Аудитория **< 1000 чел.** → один список = одна рассылка → берём цифры напрямую.
> - Аудитория **> 1000 чел.** → список дробится на несколько рассылок (бывает 10–11).
>   Чтобы получить корректные цифры по активности, показатели **складывают/усредняют
>   вручную** — проценты усредняются: `(OR₁ + OR₂ + …) / N`, абсолютные суммируются.

### 4.2. Яндекс.Метрика — для Визитов / Кликов / Целей / Оплат (все каналы с переходом на сайт)
1. `metrika.yandex.ru` → счётчик gogolschool → **Отчёты**.
2. Отчёт по **UTM-меткам** (Источники → Метки UTM), период = дата активности.
3. Найти строку по метке активности (напр. `dashapytaetsyaponyat`). Снять:
   - **Визиты из Метрики** ← колонка «Визиты»
   - **Клики** ← переходы по метке
   - **Достижения целей** ← колонка «Достижения целей»
   - **Оплаты, шт / ₽** ← по целевому действию (оплата), при необходимости сверять с CRM

### 4.3. TGStat / Telegram — для tg-каналов и посевов
1. `tgstat.com/ru/channel/@<канал>` → открыть нужный пост.
2. Снять:
   - **Охват** ← 👁 просмотры публикации
   - **Взаимодействия** ← репосты + комментарии + реакции (пример: 6 + 4 + 16 = **26**)
   - **ER / ERR** ← блок «вовлечённость по взаимодействиям»
   - подписчики канала — для расчёта **Open Rate** = Охват ÷ подписчики (340 / 2938 ≈ 11,6%)
3. **Бюджет** (для посевов) ← из чата с админом канала («Стоимость: 8 000 ₽»).

### 4.4. Instagram Insights — для inst-* (полу-ручной путь «скрин → Claude»)
Полная автоматизация через Graph API не оправдана (Meta в РФ: геоблок, App Review,
ToS/санкционные риски бана, хрупкость). Поэтому используем промежуточный вариант — без API:
1. Сотрудник открывает статистику в приложении Instagram и делает **скриншот**:
   - **пост:** Охват, Взаимодействия (репосты / отправки / сохранения);
   - **сторис:** через **Архив историй** → Охват, **клики по ссылке**.
2. Скриншот отправляется **Claude** (в чат). Claude **считывает цифры с картинки** и пишет
   их в карточку Активности (Охват, Взаимодействия, Клики).

Почему так безопасно: нет API, прокси и обхода ToS — обрабатывается только присланное
изображение. Ручным остаётся лишь сам скрин; ввод цифр и запись в Notion берёт на себя Claude.

### 4.5. Бюджет
- Платные размещения/посевы — стоимость из чата с площадкой или из договорённости.

## 5. Общие правила

### 5.1. Что не трогать
**CR клики, CR оплаты, ROAS** — формулы, считаются автоматически.

### 5.2. Open Rate по типам каналов
- **mail**: прочитано ÷ доставлено (берётся из Unisender напрямую).
- **TG**: Охват ÷ подписчики канала.

### 5.3. Несколько UTM
В поле `UTM` помещается **одна** ссылка. Если меток несколько — остальные кладутся
**в комментарии** к карточке активности (несколько ссылок в одно url-поле не влезает).

### 6.2. Unisender — статистика кампаний (✅ добавлено 23.06.2026)

В MCP-сервер `unisender` (исходники: `~/UnisenderMCP`, деплой: systemd
`unisender-mcp` на `ozma.gogol.school`) добавлены 3 инструмента:

| Инструмент | Метод API | Назначение |
|---|---|---|
| `list_campaigns(date_from, date_to, limit, offset)` | `getCampaigns` | найти рассылку по дате/теме, получить её `id` |
| `get_campaign_stats(campaign_id)` | `getCampaignCommonStats` | сводка + готовые проценты |
| `get_visited_links(campaign_id, group)` | `getVisitedLinks` | переходы по ссылкам/UTM |

`get_campaign_stats` возвращает блок **`computed`** с готовыми значениями для таблицы:
- `reach` → **Охват**
- `open_rate_pct` → **Open Rate, %**
- `click_rate_pct` → **Click Rate, %**

> Новые тулзы появятся в **новой** сессии Claude Code (список MCP-инструментов
> фиксируется на старте сессии).
>
> Нюанс: `getCampaigns` отдаёт кампании по возрастанию даты — для недельного сбора
> задавайте `date_from`/`date_to`. Дробление списков >1000 (§4.1) по-прежнему
> требует суммирования по нескольким `campaign_id`.

### 6.3. Яндекс.Метрика — UTM-отчёты (✅ добавлено 23.06.2026)

Сервер `yandex-metrika-mcp` (опенсорс `yandex-metrika-mcp-server`,
`/opt/yandex-metrika-mcp` на `ozma.gogol.school:8003`, per-request OAuth) штатно
**не умел UTM**. Дописаны 2 инструмента:

| Инструмент | Назначение |
|---|---|
| `get_utm_report(counter_id, date_from, date_to, filter, goal_id, …)` | визиты/посетители/отказы (+ достижения цели) с группировкой по UTM Source/Medium/Campaign/Content/Term; фильтр по метке активности |
| `get_custom_report(counter_id, query)` | произвольный запрос к Stat API (любые metrics/dimensions/filters) |

Маппинг в «Активности»: `ym:s:visits` → **Визиты из Метрики** и **Клики** (по метке);
достижения цели → **Достижения целей**; оплаты — через цель оплаты (`goal_id`) или CRM.

Фильтр по одной активности, пример: `filter = ym:s:UTMCampaign=='dashapytaetsyaponyat'`.

> **Осталось 2 шага до использования из Claude:**
> 1. Добавить сервер в `.mcp.json` (endpoint `http://ozma.gogol.school:8003/mcp`,
>    заголовок `Authorization: Bearer <OAuth-токен Яндекса>`).
> 2. Получить **OAuth-токен** с правом `metrika:read` (oauth.yandex.ru) — это шаг владельца.
>
> Счётчик gogolschool: **58973071** (виден в URL отчёта Метрики).

### 6.4. Рекомендованный порядок автоматизации
1. **Сначала Я.Метрика** — закрывает Визиты/Клики/Цели/Оплаты для ВСЕХ каналов сразу (наибольший охват пользы).
2. **Затем Unisender stats** — закрывает `mail` целиком (вместе с Метрикой).
3. **Telegram MCP** — уже готов, подключить логику Охват/Взаимодействия для своих каналов.
4. **TGStat API** — для посевов во внешних каналах (если их много).
5. **Instagram** — без API: путь «скрин → Claude» (§4.4).

После шагов 1–3 Claude сможет автономно заполнять `mail`, `bot-*` и свои `tg-*` активности;
для Instagram — полу-авто (скрин → Claude), полностью ручными остаются только посевы во внешних каналах.

### 6.5. Что Claude уже может делать прямо сейчас
- Прочитать список активностей в статусе «опубликовано» без метрик (`notion-query-data-sources`).
- Для **своих TG-каналов**: вытащить просмотры и реакции поста через `telegram_gogolschool` и записать Охват/Взаимодействия в Notion.
- Записать любые переданные вручную цифры в нужные поля (`notion-update-page`).
- Посчитать недостающее по формулам (Open Rate = Охват/подписчики и т.п.) перед записью.

## 7. Чек-лист недельного сбора (для человека и для агента)
1. Открыть Активности, отфильтровать: статус = «опубликовано», метрики пустые, **с даты публикации прошло ~3 дня**.
2. Для каждой строки определить `Тип канала` → источник (см. §3).
3. Снять цифры из источника (§4), учесть дробление списков Unisender (§4.1) и UTM (§5.3).

5. Перевести статус в «посчитано».
